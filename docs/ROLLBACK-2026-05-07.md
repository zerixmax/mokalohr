## 🔴 Problem

Nakon implementacije ePodrum Admin Panela (v3.0.0), sustav je počeo puca:

1. **`AdminLayout.astro`** koristio je `Layout.astro` (frontend layout) koji pokušava učitati vina iz CMS-a
2. Kada je CMS down → Navigation komponenta puca → cijela stranica se ruši
3. **Middleware** interferira s frontend routingom
4. Odvojene admin rute narušavaju headless arhitekturu

**Zaključak:** Dodavanje novog admin sučelja umjesto korištenja postojećeg Payload CMS sučelja je:
- Narušilo headless princip
- Zakompliciralo sustav
- Uvelo točke loma (CMS dependency u frontend layoutu)
